# 📚 Coding TEST(BAEKJUN) STUDY README

본 문서는 코딩테스트(백준) 학습 스터디 운영 규칙, 문제 풀이 방식, 리뷰 프로세스를
정리한 가이드 입니다.

실력 향상과 규칙적인 학습을 위해 스터디는 주 3회 정기적으로 운영됩니다.

---

## 📌 공통 규칙
```
- 풀이 언어는 Java로 통일한다
- 프로젝트 구조 및 제출 형식은 팀 합의에 따른다
- 코드 컨벤션은 Java 기본 스타일 + 개인 스타일 허용 범위 내에서 조정한다.

```

---

## 📅 스터디 일정

| 요일 | 시간 |
|------|-------|
| **화 / 목 / 토** | **08:00 ~ 10:00** |

- 총 2시간 진행  
  - **08:00 ~ 09:00** : 문제 풀이  
  - **09:00 ~ 10:00** : 코드 리뷰  

---

## 🧠 스터디 진행 방식

### 1) 08:00 ~ 09:00 — 문제 풀이
- 백준 그룹에 사전에 지정된 문제 풀이  

### 2) 09:00 ~ 10:00 — 코드 리뷰
- 양쪽 모두 풀이한 문제를 중심으로 리뷰 진행  
- 지난 시간에 리뷰하지 못한 문제도 이어서 진행  
- 리뷰 주요 포인트  
  (아직 미정)
  
### 3) 과제 규칙
- 스터디 시간 내 풀지 못한 문제는 **다음 스터디 전까지 풀이해오기**  
- 필요 시 간단한 풀이 메모 또는 복기 내용 작성  


## 🔧 자동화 관련 (선택 사항)

아직 확정되지 않았으며, 추후 스터디에서 선택 예정입니다.

가능한 옵션은 다음과 같습니다:

1) **백준 → GitHub 자동 업로드 도구 사용**  
2) **직접 풀이 파일을 업로드하는 방식 유지**

> 최종 선택은 스터디 운영 효율을 기준으로 결정합니다.
